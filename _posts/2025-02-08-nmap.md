			<p style="text-align:center; font-size:20pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">NMAP</span>
			</p>
			<p>
				&#xa0;
			</p>
			<p>
				&#xa0;
			</p>
			<p style="font-size:11pt">
				<span style="font-family:Calibri">«</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» est un puissant outil d’analyse réseau.</span>
			</p>
			<p style="font-size:11pt">
				<span style="font-family:Calibri">Ci-après, un aperçu approfondi de l’analyse avec «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">».</span>
			</p>
			<p style="font-size:11pt">
				&#xa0;
			</p>
			<p style="font-size:11pt">
				&#xa0;
			</p>
			<p style="font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="font-size:14pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">Introduction</span>
			</p>
			<p style="font-size:11pt">
				<span style="font-family:Calibri; color:#92cddc">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Dans le hacking, le pouvoir est la connaissance.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lorsque l’on dispose d’une adresse IP sur laquelle effectuer un audit de sécurité, il peut être intéressant d’avoir une idée du «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">paysage</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» à attaquer. Il faut alors définir quels services fonctionnent sur les cibles (serveur web, contrôleur de domaine Windows Active Directory…).</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cette première étape est appelée «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">le scan des ports</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">».</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Pour effectuer des requêtes sur un réseau et pour faire fonctionner plusieurs services, un ordinateur dispose de plusieurs ports réseaux pour recevoir les connexions.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Les connexions réseaux sont établies entre deux ports</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">: un port ouvert en écoute sur le serveur et un port sélectionné au hasard sur l’ordinateur client.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Chaque ordinateur dispose de 65535 ports disponibles. Cependant, bon nombre d’entre eux sont enregistrés comme ports standards</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">:</span>
			</p>
			<ul class="awlist1" style="margin:0pt; padding-left:0pt">
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>HTTP sur le port 80,
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>HTTPS sur le port 443,
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>NETBIOS sur le port 139,
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>SMB sur le port 445,
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>…
				</li>
			</ul>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Toutefois, il est possible que ces ports standards soient modifiés et il est donc important de réaliser une énumération des ports sur la machine cible.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Une analyse de ports peut être réalisée en utilisant l’outil «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">». «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» peut être utilisé pour effectuer différents types d’analyse de ports. Quoiqu’il en soit, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» se connectera tour à tour à chaque port de la machine cible. Selon la façon dont le port répond, il peut être ouvert, fermé ou filtré (par un pare-feu). Lorsque les ports sont ouverts, il est possible d’énumérer les services exécutés sur ceux-ci. </span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">«</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» est un outil puissant doté d’un moteur de scripts utile pour rechercher des vulnérabilités et dans certains cas, exécuter l’exploit directement.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>

			<p style="text-align:justify; font-size:14pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">Les différents modes «</span><span style="font-family:'Palatino Linotype'; color:#92cddc">&#xa0;</span><span style="font-family:'Palatino Linotype'; color:#92cddc">NMAP</span><span style="font-family:'Palatino Linotype'; color:#92cddc">&#xa0;</span><span style="font-family:'Palatino Linotype'; color:#92cddc">»</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri; color:#92cddc">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Comme la plupart des outils de pentest, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» s’utilise depuis un terminal de commandes. Il existe une version de «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» pour Windows et pour Linux.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify">
				<span style="font-family:Calibri; font-size:11pt">Pour démarrer l’outil «</span><span style="font-family:Calibri; font-size:11pt">&#xa0;</span><span style="font-family:Calibri; font-size:11pt">NMAP</span><span style="font-family:Calibri; font-size:11pt">&#xa0;</span><span style="font-family:Calibri; font-size:11pt">», saisir «</span><span style="font-family:Calibri; font-size:11pt">&#xa0;</span><span style="font-family:Consolas">nmap</span><span style="font-family:Consolas">&#xa0;</span><span style="font-family:Consolas">»</span><span style="font-family:Calibri; font-size:11pt"> dans un terminal de commandes. Il est ensuite possible de compléter la commande en ajoutant des arguments.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">

				<span style="font-family:Calibri">Pour afficher l’aide relative à l’outil «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">», saisir la commande « </span><span style="font-family:Consolas">nmap -h</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">».</span>

			</p>

				<span style="font-family:Calibri">«</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» essaie de se connecter sur chaque port TCP et détermine alors quel service est en cours sur le port en fonction de la réponse qu’il reçoit.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Si «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» envoie une requête TCP avec un flag SYN à un port fermé, la machine cible lui répondra avec un paquet TCP avec un RST flag.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Avec cette réponse, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» en déduit que le port est fermé.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Si, toutefois, la requête est envoyée à un port ouvert, la machine cible répondra avec un paquet TCP accompagné des SYN/ACK flags.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">«</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» relève alors le port comme étant ouvert (et il complète le handshake en renvoyant un paquet TCP avec un flag ACK).</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Il existe une dernière possibilité</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">: un port ouvert mais protégé par un pare-feu.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">La plupart des pares-feux sont configurés pour supprimer les paquets entrants.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">«</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» envoie une requête TCP SYN mais ne reçoit rien en retour.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cela indique que le port est protégé par un pare-feu et il est considéré comme filtré.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cela dit, il est facile de configurer un pare-feu pour qu’il réponde avec un paquet TCP RST.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Pour un pare-feu IPtables pour Linux, il suffirait d’entrer la commande suivante</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">:</span>
			</p>
			<table style="border:0.75pt solid #000000; padding:0pt; border-collapse:collapse">
				<tr>
					<td style="width:442.3pt; padding:0pt 5.03pt; vertical-align:top; background-color:#000000">
						<p style="text-align:justify; font-size:11pt">
							<span style="font-family:Consolas; color:#ffffff">iptables -I INPUT -p tcp –dport &lt;port&gt; -j REJECT –reject-with tcp-reset</span>
						</p>
					</td>
				</tr>
			</table>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cette commande rendrait alors difficile (voire impossible) d’obtenir une lecture précise de la cible.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:14pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">SYN Scans</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Comme pour les analyses TCP, les analyses SYN (-sS) sont utilisées pour analyser la plage de ports TCP d’une ou plusieurs cibles</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">; cependant ces 2 types d’analyse fonctionnent légèrement différemment. Les analyses SYN sont parfois appelées analyses «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">à moitié ouvertes</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» ou analyses «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">furtives</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">».</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lorsque les analyses TCP effectuent une négociation three-way handshake avec la cible, les analyses SYN renvoient un paquet TCP RST après avoir reçu un SYN/ACK du serveur (cela empêche le serveur d’essayer à plusieurs reprises de faire la demande).</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cela présente de nombreux avantages pour les attaquants</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">:</span>
			</p>
			<ul class="awlist5" style="margin:0pt; padding-left:0pt">
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>Il peut être utilisé pour contourner les anciens systèmes de détection d’intrusion car ils recherchent une négociation three-way handshake. Ce n’est plus le cas avec les solutions IDS modernes&#xa0;; c’est pour cette raison que les analyses SYN sont qualifiées d’analyses «&#xa0;furtives&#xa0;».
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>Les analyses SYN ne sont souvent pas enregistrées par les applications écoutant sur les ports ouverts, car la connexion est enregistrée une fois la connexion établie.
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>Sans se soucier d’effectuer une négociation three-way handshake pour chaque port, les analyses SYN sont nettement plus rapides qu’une analyse TCP standard.
				</li>
			</ul>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Les analyses SYN présentent également certains inconvénients</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">:</span>
			</p>
			<ul class="awlist6" style="margin:0pt; padding-left:0pt">
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>Ils nécessitent des autorisations sudo pour fonctionner correctement sous Linux. En effet, les analyses SYN nécessitent la possibilité de créer des paquets bruts, ce qui est un privilège que seul l’utilisateur root possède par défaut.
				</li>
				<li class="ListParagraph" style="text-indent:-18pt; text-align:justify; font-family:Calibri; font-size:11pt">
					<span style="width:14.63pt; font:7pt 'Times New Roman'; display:inline-block">&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0;&#xa0; </span>Les services instables sont parfois interrompus par les analyses SYN, ce qui peut s’avérer problématique si un client a fourni un environnement de production pour le test.
				</li>
			</ul>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Dans l’ensemble, les avantages l’emportent sur les inconvénients.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Pour cette raison, les analyses SYN sont les analyses par défaut utilisées par «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» si elles sont exécutées avec les autorisations sudo. S’il est exécuté sans autorisations sudo, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» utilise par défaut l’analyse TCP.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; border-bottom:0.75pt solid #000000; padding-bottom:1pt; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lorsqu’on utilise une analyse SYN pour identifier les ports fermés et filtrés, les mêmes règles que pour une analyse TCP s’appliquent.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Si un port est fermé, le serveur répond avec un paquet TCP RST. Si le port est filtré par un pare-feu, le paquet TCP SYN est soit supprimé, soit usurpé avec une réinitialisation TCP.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<div style="border-bottom:0.75pt solid #000000; clear:both">
				<p style="text-align:justify; font-size:11pt">
					<span style="font-family:Calibri">Les deux analyses sont identiques</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">: la grande différence réside dans la manière dont elles gèrent les ports ouverts.</span>
				</p>
				<p style="text-align:justify; padding-bottom:1pt; font-size:11pt">
					<span style="font-family:Calibri">&#xa0;</span>
				</p>
			</div>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:14pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">UDP Scans</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri; color:#92cddc">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Contrairement à TCP, les connexions UDP sont sans état. Cela signifie que, plutôt que d’initier une connexion avec une négociation handshake aller-retour, les connexions UDP reposent sur l’envoi de paquets vers un port cible et essentiellement sur l’espoir qu’ils y parviennent. Cela rend UDP idéal pour les connexions qui reposent sur la vitesse plutôt que sur la qualité (par exemple, le partage vidéo), mais le manque de reconnaissance rend l’analyse d’UDP beaucoup plus difficile (et beaucoup plus lente). Le commutateur pour une analyse «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» UDP est (-sU).</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lorsqu’un paquet est envoyé vers un port UDP ouvert, il ne devrait y avoir aucune réponse. Lorsque cela se produit, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» fait référence au port comme étant ouvert/filtré. En d’autres termes, il soupçonne que le port est ouvert, mais potentiellement protégé par un pare-feu. S’il obtient une réponse UDP (ce qui est très inhabituel), alors le port est marqué comme ouvert. Le plus souvent, il n’y a pas de réponse, auquel cas la demande est envoyée une seconde fois à titre de double vérification. S’il n’y a toujours pas de réponse, le port est marqué ouvert/filtré et «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» continue.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lorsqu’un paquet est envoyé vers un port UDP fermé, la cible doit répondre avec un paquet ICMP (ping) contenant un message indiquant que le port est inaccessible. Cela identifie clairement les ports fermés, que «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» marque comme tels et continue.</span>
			</p>
			<div style="border-bottom:0.75pt solid #000000; clear:both">
				<p style="text-align:justify; font-size:11pt">
					<span style="font-family:Calibri">&#xa0;</span>
				</p>
				<p style="text-align:justify; padding-bottom:1pt; font-size:11pt">
					<span style="font-family:Calibri">&#xa0;</span>
				</p>
			</div>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">En raison de cette difficulté à identifier si un port UDP est réellement ouvert, les analyses UDP ont tendance à être incroyablement lentes par rapport aux différentes analyses TCP (de l’ordre de 20 minutes pour analyser les 1000 premiers ports, avec une bonne connexion). Pour cette raison, il est généralement recommandé d’exécuter une analyse «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» avec –top-ports &lt;number&gt; activé. Par exemple, une requête nmap -sU –top-ports 20 &lt;target&gt; analysera les 20 ports UDP les plus couramment utilisés, ce qui entraînera un temps d’analyse beaucoup plus acceptable. </span>
			</p>
			<p style="text-align:justify; border-bottom:0.75pt solid #000000; padding-bottom:1pt; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lors de l’analyse des ports UDP, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» envoie généralement des requêtes complètement vides</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">: juste des paquets UDP bruts. Cela dit, pour les ports qui sont habituellement occupés par des services connus, il enverra à la place une charge utile spécifique au protocole qui est plus susceptible de susciter une réponse à partir de laquelle un résultat plus précis pourra être tiré.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>

			<p style="text-align:justify; font-size:14pt">
				<span style="font-family:'Palatino Linotype'; color:#92cddc">NULL, FIN and Xmas</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Les analyses de ports TCP NULL, FIN et Xmas sont moins couramment utilisées que toutes les autres analyses précédemment abordées. Ces 3 analyses sont liées et sont utilisées principalement car elles sont encore plus furtives, qu’un scan furtif SYN.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">1/ NULL scans</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Comme le nom l’indique, les analyses NULL (-sN) se produisent lorsque la requête TCP est envoyée sans aucun indicateur défini. L’hôte cible doit alors répondre par un RST si le port est fermé.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">2/ FIN scans</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Les analyses FIN (-sF) fonctionnent de manière presque identique</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">; cependant, au lieu d’envoyer un paquet complètement vide, une requête est envoyée avec l’indicateur FIN (généralement utilisé pour fermer une connexion active). Encore une fois, «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» attend un RST si le port est fermé.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">3/ Xmas scans</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Comme pour les 2 autres analyses de cette classe, les analyses Xmas (-sX) envoient un paquet TCP mal formé et attendent une réponse RST pour les ports fermés. On l’appelle analyse de Noël (Xmas) car les indicateurs qu’elle définit (PSH, URG et FIN) lui donnent l’apparence d’un arbre de Noël clignotant lorsqu’elle est vu dans une capture de paquet dans Wireshark.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">La réponse attendue pour les ports ouverts avec ces analyses est également identique et très similaire à celle d’une analyse UDP. Si le port est ouvert, il n’y a aucune réponse au paquet mal formé. Malheureusement (comme pour les ports UDP ouverts), c’est également un comportement attendu si le port est protégé par un pare-feu, donc les analyses NULL, FIN et Xmas identifieront uniquement les ports comme étant ouverts/filtrés, fermés ou filtrés. Si un port est identifié comme filtré avec l’une de ces analyses, c’est généralement parce que la cible a répondu avec un paquet ICMP inaccessible. </span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Il est important de noter que même si la RFC 793 exige que les hôtes du réseau répondent aux paquets mal formés avec un paquet TCP RST pour les ports fermés, et ne répondent pas du tout sur les ports ouverts</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">; ce n’est pas toujours le cas dans la pratique. En particulier, Microsoft Windows (et de nombreux périphériques réseau Cisco) sont connus pour répondre par un RST à tout paquet TCP mal formé, que le port soit réellement ouvert ou non. De ce fait, tous les ports apparaissent comme étant fermés.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Cela étant dit, le but est bien de contourner le pare-feu. De nombreux pare-feu sont configurés pour rejeter les paquets TCP entrants vers des ports bloqués sur l’indicateur SYN est défini (bloquant ainsi les nouvelles demandes d’initiation de connexion). En envoyant des requêtes qui ne contiennent pas le flag SYN, le pare-feu est ainsi efficacement contourné. Cependant, la plupart des solutions IDS modernes maîtrisent ce type d’analyse.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>

			<p style="text-align:justify; font-size:11pt">

				<span style="font-family:'Palatino Linotype'; color:#b6dde8">ICMP Network Scanning</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Lors de la première connexion à un réseau cible, le premier objectif est d’obtenir une carte de la structure du réseau – c’est-à-dire connaître les adresses IP correspondantes à des hôtes actifs et celles qui n’en n’ont pas.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Le moyen de faire cela est d’utiliser «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» pour effectuer un ping scanning. «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» envoie un paquet ICMP à chaque adresse IP possible pour le réseau spécifié. Lorsqu’il reçoit une réponse, il marque l’adresse IP qui a répondu comme étant active. Cela n’est pas toujours exact</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">; cependant, il peut fournir une référence et mérite d’être réalisé.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">Pour effectuer un ping sweep, il faut utiliser le commutateur -sn suivi des plages IP qui peuvent être spécifiées avec un trait d’union (-) ou une notation CIDR. Exemple</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">: pour analyser le réseau 192.168.0.x, on utilise les commandes</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">:</span>

			</p>

				<span style="font-family:Calibri">L’hôte Windows de base, avec le pare-feu par défaut, bloquera tous les paquets ICMP. Cela pose problème car le ping est souvent utilisé pour s’assurer qu’un hôte est en ligne, ce que fait «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">». «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» considérera cet hôte comme étant hors-ligne et il ne prendra pas la peine de l’analyser.</span>
			</p>
			<p style="text-align:justify; font-size:11pt">
				<span style="font-family:Calibri">&#xa0;</span>

Dans le cas d’un hôte «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">mort</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">», «</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">NMAP</span><span style="font-family:Calibri">&#xa0;</span><span style="font-family:Calibri">» vérifiera et revérifiera toujours chaque port.</span>
